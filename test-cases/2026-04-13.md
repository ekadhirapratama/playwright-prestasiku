### Interpretasi kolom Checklist (TRUE/FALSE)

Nilai **Checklist** di CSV adalah **status pengujian oleh QA tim lain**, bukan definisi fitur:

| Nilai | Arti | Implikasi untuk dokumen ini |
| --- | --- | --- |
| **TRUE** | Skenario task tersebut **sudah pernah ditest** oleh QA tim lain | Bisa dipakai sebagai referensi regresi cepat; tidak wajib jadi prioritas uji ulang kecuali ada perubahan modul. |
| **FALSE** | Skenario task tersebut **belum ditest** (atau belum dilaporkan selesai) oleh QA tim lain | **Acuan test case utama** untuk tim ini: setiap baris Task di bawah dengan FALSE = kandidat kasus uji yang perlu dirancang/dieksekusi. |

Poin-poin di kolom **Task** (nomor urut + teks) di bawah menyalin struktur CSV; gunakan sebagai daftar skenario yang harus dicakup untuk modul terkait.

## Langkah-langkah testing (umum)

1. Buka `https://prestasikukaigroup-alpha.kai.id/nipp?nipp={NIPP}` — tutup dialog SweetAlert jika muncul (**OK**).  
2. Untuk **Penilaian Semester 1 (pribadi):** sidebar → expand **Pengajuan Penilaian Kerja** → klik **Pribadi** — pastikan URL `/penilaian-kinerja/pribadi`, judul halaman konsisten, dan tabel/aksi **Penilaian** (mis. semester) dapat diakses.  
3. Untuk **Grade 6–8 / 9–18:** gunakan NIPP dari tabel referensi `masternipp.json` di bagian 1; verifikasi form/field sesuai grade bila menguji item checklist terkait.  
4. Untuk **Kepala Daerah / Direksi / Korporasi:** **Upload Kontrak Kinerja** → pilih jalur sesuai backlog E6.  
5. Untuk **Koreksi / Revisi:** **Approval** → **Revisi Penilaian** atau **Perubahan atasan** — sesuaikan dengan skenario backlog E2.S12+ / E3.S5+.  
6. Untuk **hasil SMT 1 ter-approve:** **Kesepakatan Kinerja** → **Pribadi** → pilih tahun → tab/section penilaian SMT 1 (E8.S3).  
7. Untuk **Penilaian Bawahan:** **Pengajuan Penilaian Kerja** → **Bawahan** — verifikasi daftar bawahan/vacant/cuti/izin sesuai data.  
8. Untuk **Mutasi pasca perencanaan:** gunakan NIPP mutasi (40404 / 40826 / 74654); pada **Pengajuan Penilaian Kerja** → **Pribadi**, verifikasi perilaku Mutasi; tambahkan **Laporan** → **Mutasi** / **Pensiun** bila menguji pelaporan.  
9. Dokumentasikan screenshot URL, NIPP, dan status baris data (draft, active, mutation).
